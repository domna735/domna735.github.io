即時面部表情辨識系統（Real-Time FER）

PyTorch · OpenCV · ONNX Runtime · MobileNetV3 · YuNet · CLAHE · KD/DKD

  • 建立 7 類表情辨識流程（pipeline），提升少數類別（Disgust、Fear、Sad）之辨識表現。
  • 設計教師模型—學生模型蒸餾（Teacher–Student；RN18／B3／CNXT → MobileNetV3），Macro-F1 ≈ 0.79，驗證集準確率約 80%。
  • 建置多執行緒即時推論流程（擷取 → 前處理 → 推論 → 平滑），CPU 可達 25–30 FPS。
  • 透過時間序列穩定化（EMA、hysteresis、投票視窗）降低輸出抖動（flicker），提升使用體驗。
  • 提供可重現之機器學習流程：資料清理 → 清單（manifest）驗證 → 訓練 → 匯出 → 部署。

  • 由零開始建立可即時運行的多模態互動原型，整合表情辨識、對話生成與語音輸出至單一應用。

UGOT 機械人比賽訓練（七堂課完整課程設計）

涵蓋工程基礎、夾取操作、拋投機制、以及比賽策略與臨場演練。

Lesson 1 — 認識工程車（自由探索）

安全規則 · 基線評估 · 學習動機

  • 課堂目標：讓學生熟悉工程車操作；觀察協調能力與程度差異；建立課堂氣氛。
  • 課堂流程：安全規則與基本操作（約 5 分鐘）→ 自由探索（約 20 分鐘）→ 小挑戰：到達指定位置（約 10 分鐘）→ 總結（約 5 分鐘）。
  • 老師重點：不急於教授過多內容；以觀察與引導為主，辨識需要支援及可提供進階挑戰的學生。

Lesson 2 — 工程車移動（Motion 基礎）

X/Y/Z · Motion Blocks · Checkpoints

  • 課堂目標：理解 X/Y/Z 及 Motion Block 控制概念；完成「可見成果」（到達檢查點）。
  • 任務設計：前進 50 cm → 右轉 90° → 到達 checkpoint 2 → 左移到 checkpoint 3，全程避免碰撞。
  • 老師重點：以地面膠紙設 checkpoint；將概念講清楚、講得簡單，並讓學生透過調速獲得成功感。

Lesson 3 — 工程車夾取操作（方塊／球）

J1/J2/J3 · Gripper · Pick → Place

  • 課堂目標：掌握手臂關節（J1/J2/J3）與夾爪開合；完成「夾起 → 放置」任務。
  • 任務設計：手臂下降 → 夾起目標物 → 放至指定位置 → 減少掉落與碰撞。
  • 老師重點：避免一次過灌輸三軸概念，採用逐軸教學；鼓勵學生「試錯—修正」以建立除錯能力。

Lesson 4 — 拋投（拋球／投擲機制）

角度控制 · Wait 時間調整 · 安全管理

  • 課堂目標：理解「角度 + Wait」對力度的影響；完成 2–3 種不同力度的拋投。
  • 任務設計：低拋（20–40 cm）／中拋（50–80 cm）／高拋（越過障礙物）。
  • 老師重點：以安全為先；引導學生採用「嘗試 → 調整 → 再嘗試」迭代，找出最佳角度與力度組合。

Lesson 5 — 策略課（比賽策略）

規則分析 · 分工 · 時間管理

  • 課堂目標：把比賽規則轉化為可執行策略（先後次序、風險控制、避免犯規）。
  • 小組活動：2–3 人一組設計比賽流程並畫出策略；老師逐組提供回饋。
  • 老師重點：策略重在具體可操作（時間與路線、分工與節奏），而非抽象口號。

Lesson 6 — 補課／練習／Debug（Buffer）

追進度 · 大量練習 · 友誼賽

  • 課堂目標：鞏固前面內容；為程度較慢的學生提供追趕時間，同時給予進度較快的學生額外挑戰。
  • 活動：夾球／拋球／行 checkpoint 小比賽；非正式小組對戰與除錯時間。
  • 老師重點：此堂為「支援與收斂」課，重點是讓全班能力拉齊並建立信心。

Lesson 7 — 最終作戰（正式比賽模擬）

合作 · 臨場表現 · 反思總結

  • 課堂目標：把所有技能串連為完整比賽流程；訓練合作與臨場冷靜執行。
  • 流程：2 人一隊 → 計時比賽 → 計分 → 小獎勵 → 反思與總結（強調進步而非勝負）。
  • 老師重點：營造正向氛圍，避免過度競爭；讓學生感受到「自己做得到」。

教學重點

  • 循序漸進：基礎 → 操作 → 拋投 → 策略（由技能到整合）。
  • 以 UCode4 為主之實作教學：清晰視覺化投影片、逐步示例、課堂小挑戰。
  • 針對混合能力班別：提供可達成的階段目標與充足迭代／除錯時間。
  • 強調工程思維：拆解問題、迭代驗證、紀律化除錯與改進。
