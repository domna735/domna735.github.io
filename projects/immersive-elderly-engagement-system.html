Key project

Immersive Elderly Engagement System (XR)

A service-learning XR deployment combining 360° virtual walks and photogrammetry / point-cloud reconstructions, delivered through a Joy-Con controlled Unity application for elderly engagement and digital inclusion.

GitHub repo Materials (on request) Back to projects
Why it matters: The goal is not “XR for XR’s sake” — it is to design an interaction that elderly users can learn quickly, and to use familiar places and culturally resonant scenes to support conversation, nostalgia, and well-being.
Note: due to dataset size and on-site privacy constraints, the public repo may not include all media / 3D assets.

Executive summary

360° virtual walk + 3D reconstruction + accessible interaction.

This project designs and deploys an immersive digital experience for underprivileged elderly residents in Yunnan, combining:

  • 360° virtual walks: video-based exploration of culturally significant locations.
  • High-fidelity 3D reconstruction: photogrammetry / point-cloud assets (e.g., .ply) of familiar spaces.
  • Unity interaction with Joy-Con: intuitive navigation and motion-driven feedback.

During on-site deployment and user testing, we observed increased emotional engagement, nostalgia, and social interaction among participants — highlighting how immersive technologies can support well-being, memory, and digital inclusion.

System overview

Designed for users with limited digital literacy.

User flow

  1. Main menu: browse destinations (each destination maps to a video clip / scene).
  2. Experience scene: play the selected 360° walk with an overlay UI (progress, distance, step count).
  3. Settings panel: pause/resume, return to menu, and adjust sensitivity.

Interaction design goals

  • Repeatable controls: simple button and stick inputs; consistent mapping across scenes.
  • Activity-driven feedback: video playback can respond to motion (IMU gyro magnitude).
  • Accessibility: large, readable UI and minimal cognitive load.

Controls (Joy-Con)

Designed for learnability and low error-rate.

Context Input Action
Main menu D-Pad Left / Right Previous / next destination
Main menu Shoulder button (confirm) Enter experience scene with selected clip
Experience Stick Rotate camera (yaw / pitch)
Experience IMU motion (gyro magnitude) Auto play/pause based on activity threshold
Experience Shoulder button Open settings panel (pauses video)
Settings panel D-Pad Down Close settings panel (resumes video)
Settings panel D-Pad Up Return to main menu

A mouse fallback can be used during development; the primary deployment target is Joy-Con.

Content pipeline

Capture → reconstruction → Unity integration.

  • 360° capture: on-site 360 camera capture (Insta360) plus supplementary aerial footage where available.
  • 3D reconstruction: photogrammetry tooling (e.g., LumaAI) to produce reconstructions and point clouds.
  • Unity integration: selectable 360 clips, UI overlays, and interaction logic.
  • Optional advanced rendering: point-based / splatting-style rendering is a future path for higher fidelity 3D viewing.

The system intentionally keeps the user-facing experience simple while leaving a path to incorporate richer 3D assets as device capability and deployment constraints allow.

On-site deployment & outcomes

Qualitative results from service-learning deployment.

Observed outcomes

  • Initial hesitation shifted to curiosity and enjoyment after brief acclimatisation.
  • Participants showed nostalgia when recognising familiar environments (a strong engagement trigger).
  • Sessions often became social: users shared stories and memories while exploring together.

Challenges encountered

  • Technical robustness: occasional lag / synchronisation issues can break immersion.
  • Communication barriers: varied digital literacy required more patient instruction and simpler explanations.
  • On-site constraints: space, noise, and setup logistics shaped session pacing and reliability.

Guiding theme

Empathetic technology, delivered with sincere human connection and collaborative dedication, can uplift broken spirits by bridging the digital divide.

Limitations & next steps

  • Input consistency: standardise axis mappings and consider migrating to Unity’s newer Input System.
  • Media scalability: large video datasets can bloat builds; consider Addressables / AssetBundles for scale.
  • Step detection: replace basic thresholding with filtered peak detection and per-device calibration.
  • Joy-Con reliability: improve connection UX, disconnect handling, and explicit controller selection.
  • Accessibility iteration: co-design with elderly users earlier; add larger fonts and simplified navigation where needed.

How to run (high-level)

For recruiters: steps are intentionally concise and reproducible.

  1. Open the Unity project in a compatible Unity Editor version (Unity 6 / 2022 LTS depending on branch/tooling).
  2. Load the menu scene and press Play.
  3. Pair Joy-Con(s) over Bluetooth (Windows) and verify input recognition.
  4. Select a destination to start the 360° walk experience.
