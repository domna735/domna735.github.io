重點項目

多模態虛擬溝通系統（MVP）

一個以示範為先（demo-first）的即時多模態 AI 互動原型。

GitHub 返回項目列表
目標：建立一個可運行、可展示的系統（而非只做單一模型）—— 能夠「看、聽、說、回應」，並在即時互動中維持一致的 persona 行為。

證據與公開狀態

說明現階段可公開取得的內容，以及可應要求提供的材料。

  • 現已公開： GitHub 專案 （原始碼及基本設定說明）。
  • 可應要求提供： 短示範錄影，以及（如需要）更完整的設定／配置走讀筆記，方便評估系統整合與使用流程。

概覽

以離線優先（offline-first）、模組化與 Windows 友善為設計取向（支援 DirectML 路徑）。

本 MVP 以示範可運行為優先，整合以下能力於單一應用：

  • 面部表情辨識（FER）
  • persona 感知的對話生成
  • 語音輸出（TTS）
  • 語音輸入（STT，選用）
  • 語調／聲音情緒（prosody／voice mood）分析（選用）
  • 即時 UI 叠加 + 會話日誌（session logging）

系統預設以離線方案確保可運行；在環境與金鑰允許時，可切換至雲端後端以提升互動品質（並保留失效時的降級／回退策略）。

功能

即時面部表情辨識（FER）

Webcam · Distilled ONNX student · YuNet · EMA / hysteresis / voting

  • 以 webcam 進行即時偵測；採用輕量化 ONNX 學生模型（由教師模型蒸餾）。
  • 使用 YuNet 作臉部偵測，並加入時間序列平滑（EMA／hysteresis／投票視窗）以穩定輸出。
  • UI overlay 即時顯示情緒標籤與信心度。

persona 感知對話

離線規則 · Azure OpenAI（選用）· persona 切換

  • 離線規則式代理作為預設（確保隨時可用），並提供 Azure OpenAI 後端（選用）。
  • 支援 persona 切換（例如 Friendly／Professional／Playful）。
  • 回應可同時參考情緒訊號與 persona 狀態，維持互動一致性。

語音輸出（TTS）

Windows TTS · Azure Speech TTS（選用）· 自動回退

  • 預設採用 Windows 內建 TTS，零設定、穩定可靠。
  • 支援 Azure Speech TTS（選用）以獲得較佳語音品質。
  • 線上服務失效時自動降級（fallback）至本機 TTS。

語音輸入（STT）— 選用

Azure Speech STT · Push-to-talk · 免手持連續模式

  • Push-to-talk 模式：控制性高，便於穩定示範。
  • 免手持連續收音模式：在 TTS 播放期間自動暫停收音，降低自我轉錄風險。
  • 以自然語音互動為目標，並配合日誌記錄與延遲量測。

語調／聲音情緒（Prosody / Voice Mood）— 選用

Mic capture · Pitch + energy · Mood mapping

  • 即時收音 → pitch 與能量分析（demo-level）。
  • 粗略映射至簡化標籤（例如 calm／excited／tense／sad）。
  • 與 FER 結合，用於調整對話語氣與互動風格。

日誌與評估

JSONL 會話日誌 · 延遲統計 · 可重現與除錯

  • 以 JSONL 記錄每段互動（情緒、persona、STT/LLM/TTS 延遲等）。
  • 提供延遲彙整腳本，便於快速檢視系統反應時間與穩定性。
  • 設計目標為可重現、可觀測與易於除錯。

架構

模組化流程：離線可運行；線上服務可選用並具回退策略。

Webcam → FER → Dialogue Agent → TTS → UI Overlay

Optional:
Microphone → STT → Dialogue Agent
Microphone → Prosody → Dialogue Agent

所有元件均以可替換、可擴展為目標，便於後續研究延伸（例如 expressive TTS、voiceprint、multimodal fusion）。

定位與價值

  • 核心重點在「系統整合」：把模型變成可用的即時互動系統。
  • 展示即時 ML 工程、多模態 pipeline 設計、穩健的 fallback 邏輯與 Windows 部署。
  • 透過日誌與延遲量測，強化可觀測性與可重現性，便於迭代與報告。
  • 作為後續研究的基礎平台：更穩健的 FER、聲紋／語調建模、表現力更強的語音輸出、多模態融合等。
開發紀錄（2026 年 1–2 月）— demo-first MVP 重點
  • 以可演示、可穩定運行為首要目標：FER → persona 對話 → TTS（選用 STT／prosody）。
  • 離線優先：對話與 TTS 均有本機預設；雲端後端作可選升級。
  • STT 支援 push-to-talk 與連續模式；並在 TTS 播放期間暫停 STT，降低回音與誤轉錄。
  • 以 JSONL 記錄互動與延遲，並提供彙整工具支援評估與撰寫報告。
