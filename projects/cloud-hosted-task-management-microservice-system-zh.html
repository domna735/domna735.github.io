重點項目

雲端託管任務管理微服務系統

一個可雲端部署的 Spring Boot 微服務，整合 Session 驗證、使用者任務管理與計算工具 API， 並強調可重現測試流程與部署導向工程實踐。

執行摘要

COMP4442 Group 14 · 最後更新：2026-03-29

Cloud Compute Service 將三項功能整合為一致的後端平台：Session 驗證、使用者任務 CRUD，以及計算工具 API。 系統採用分層式 Spring Boot 架構（Controller、Service、Repository、Entity）， 並在不同層次落實安全與驗證控制。

  • 驗證 API：register、login、logout、current user。
  • 任務 API：create、list、get-by-id、update、delete。
  • Compute API：service ping 與算術計算。
  • 提供靜態前端頁面支援註冊、登入及任務操作流程。
  • 整合測試 + smoke scripts，確保行為可重現與可驗證。

架構與安全模型

分層設計，並以架構層面確保使用者擁有權隔離。

Controller → Service → Repository → Database

身分來源：
Spring Security SecurityContext（Session）

擁有權傳遞：
Authenticated principal → service logic → user-scoped repository queries
  • 採用 Spring Security 的 session-based 驗證；客戶端以 JSESSIONID 維持登入狀態。
  • 密碼以 BCrypt 雜湊儲存，不保存明文密碼。
  • 任務資料查詢以使用者維度限制（例如 findByIdAndUserId），避免跨使用者讀寫。
  • 未授權 API 會回應 401；受保護頁面在未登入時會導向登入頁。

驗證與可重現性

  • 整合測試覆蓋驗證流程、受保護路由、任務 CRUD 生命週期與錯誤語義。
  • smoke scripts 提供快速 API 健康檢查，降低 demo 前回歸風險。
  • one-command 開發腳本可啟動服務、等待就緒並自動執行 smoke 驗證。
  • register 端點之 201 Created 契約已與腳本及文件對齊，避免誤報失敗。
典型 smoke 驗證流程 
Ping → Register (201) → Login → Current user
→ Create task (201) → List tasks → Logout

雲端部署準備

  • 透過 profile-based configuration 分離本地與生產環境設定。
  • 開發環境使用 H2，提升本地迭代速度。
  • 生產環境透過環境變數注入資料庫設定（EC2 + RDS 目標場景）。
  • 部署素材包含 runtime templates、verifier scripts 與 systemd unit template。

限制與後續方向

  • CSRF 策略可針對更完整生產情境再強化。
  • 以 Flyway 或 Liquibase 取代 schema auto-update。
  • 加入登入濫用防護（rate limit、lockout 等）。
  • 任務列表可加入 pagination 與篩選能力。
  • 強化可觀測性：結構化日誌、指標、追蹤與告警。
